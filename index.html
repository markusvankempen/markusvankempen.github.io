How it works

From IDE chat to live enterprise data

Every project follows the same pattern: an AI assistant calls MCP tools; the server talks to your platform APIs. Extensions add visual UIs, diagnostics, and one-click mcp.json setup.

01

Install & connect

VS Code Marketplace, Open VSX, npx, or Cursor one-click install. Credentials stay local or on Code Engine.

02

MCP tools in your IDE

Ask the AI to list backends, query Maximo, manage WxO agents, submit OpenQASM, or handle Zendesk tickets — via typed MCP tools.

03

Deploy remotely (optional)

IBM Code Engine SSE gateways share one URL across the team — server-side secrets, dashboard, and connection test UI.

About

Markus van Kempen

Based in Toronto, I build open-source tools at the intersection of enterprise platforms and AI-assisted development — bridging IBM Watsonx Orchestrate, Maximo, IBM Quantum, and Zendesk with MCP and modern IDEs.

The Agent Bridge theme reflects what these projects do: connect AI agents to systems of record through the Model Context Protocol — securely, repeatably, and with great developer ergonomics.

All projects are Apache 2.0. Contributions, issues, and feedback are welcome.
